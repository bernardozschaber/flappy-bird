\hypertarget{md_README_autotoc_md2}{}\doxysubsection{\texorpdfstring{📋 Descrição do Problema}{📋 Descrição do Problema}}\label{md_README_autotoc_md2}
O objetivo deste projeto é implementar uma versão simplificada do jogo {\bfseries{Flappy Bird}}, aliando\+:


\begin{DoxyEnumerate}
\item {\bfseries{Padrão Orientado a Objetos}} em C++, com hierarquias de classes para tela de jogo e UI. ~\newline

\item {\bfseries{Interface gráfica}} de menu (login, cadastro, listagem e remoção de jogadores) usando Allegro 5. ~\newline

\item {\bfseries{Persistência}} de cadastro de jogadores em arquivo de texto, com armazenamento de nome, senha, recorde, partidas jogadas e {\itshape conquistas}.
\item {\bfseries{Integração}} consistente entre a manipulação dos arquivos e a execução do jogo em si.
\item {\bfseries{Execução robusta}} do jogo, sem bugs, usando Allegro 5.
\item {\bfseries{Configuração personalizável}} das variáveis do jogo.
\end{DoxyEnumerate}

O sistema deve permitir\+: ~\newline

\begin{DoxyItemize}
\item {\bfseries{Login/\+Logout}} de usuários. ~\newline

\item {\bfseries{Registro}} de novos jogadores. ~\newline

\item {\bfseries{Listagem paginada}} de todos os jogadores, com destaque de recordistas e navegação entre páginas. ~\newline

\item {\bfseries{Remoção}} de jogadores, com confirmação de senha. ~\newline

\item {\bfseries{Execução do jogo}} propriamente dito (Flappy Bird).
\end{DoxyItemize}
